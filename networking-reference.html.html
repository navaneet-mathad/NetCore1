Chapter 4 · Infrastructure Design
NETWORK
TOPOLOGIES

Five fundamental arrangements of nodes and connections. Physical topology is the actual layout. Logical topology describes data flow.

BUS
LOW FAULT TOLERANCE LOW COST

All devices on a single backbone cable. Cable failure = entire network down. Obsolete since 1990s.

RING
TOKEN BASED LEGACY

Closed loop. Token controls transmission. IBM Token Ring, FDDI — legacy systems only.

STAR ✓
MOST COMMON MODERN LAN

All devices to a central switch. Easy troubleshooting. One failure doesn't affect others. Your home network.

TREE
HIGHLY SCALABLE ENTERPRISE

Hierarchical star networks. Core → floor switches → desks. Universities, enterprise campuses.

MESH
NO SINGLE POF HIGH COST

Every device connects to every other. n(n-1)/2 links. Internet backbone, military, ISP core networks.

Chapter 16 · Threat Landscape
NETWORK SECURITY
THREATS

Modern threats, attack vectors, and defensive countermeasures. Know your adversary.

🦠
RANSOMWARE
Encrypts files and demands cryptocurrency ransom. LockBit, REvil, Conti extorted $1B+ from companies. NotPetya paralysed Maersk ($300M damage).
🛡️ WildFire sandboxing + backup isolation + network segmentation
🎣
PHISHING
Email/web deception to steal credentials or install malware. BEC fraud — $26B+ per year globally. Most common initial access vector.
🛡️ URL Filtering + Email Security + User training
💉
SQL INJECTION
Database manipulation via unsanitised web input. ' OR 1=1; DROP TABLE users-- → full database access.
🛡️ WAF + App-ID + Threat Prevention signatures
🌊
DDoS ATTACK
Overwhelming target resources. Mirai botnet (500 Gbps) took down Dyn DNS 2016. Types: Volumetric, Protocol, Application-layer.
🛡️ Zone Protection Profiles + Anti-DDoS + SYN cookies
🕵️
APT / NATION STATE
Advanced Persistent Threats — stealthy long-term compromise. APT29 SolarWinds supply chain attack 2020 affected 18,000+ customers.
🛡️ Cortex XDR behavioral detection + AutoFocus threat intel
🔀
MITM ATTACK
Intercept/alter communications between two parties. ARP spoofing on LAN, BGP hijacking at internet scale.
🛡️ TLS inspection + HSTS + Dynamic ARP Inspection (DAI)

Chapter 18 · Modern Architecture
ZERO TRUST
ARCHITECTURE

"Never Trust, Always Verify" — coined by John Kindervag at Forrester Research. Assumes breach has ALREADY occurred.

01
VERIFY EXPLICITLY
Always authenticate and authorise based on ALL available data — identity, device health, location, behaviour. MFA + device trust + continuous re-authentication.
02
LEAST PRIVILEGE
Limit access to ONLY what's needed, ONLY when needed (Just-in-Time). ZTNA instead of VPN — VPN gives full network access, which is far too broad.
03
ASSUME BREACH
Design as if the network is already compromised — limit blast radius. Microsegmentation, E2E encryption, comprehensive logging, proactive threat hunting.
TRADITIONAL VPN
❌ Full network access after auth — like handing someone ALL your house keys
❌ Trust never expires during session
❌ Easy lateral movement once inside tunnel
❌ No inspection after tunnel established
❌ One compromised laptop = full network access
ZTNA 2.0 (PRISMA ACCESS)
✅ App-specific access only — only the app you're authorised for
✅ Continuously verify THROUGHOUT session — trust revocable mid-session
✅ Zero implicit trust — no lateral movement possible
✅ Full L7 inspection of ALL traffic including ZTNA connections
✅ One compromised device = only sees ONE app
